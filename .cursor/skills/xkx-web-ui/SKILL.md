# 侠客行 Web UI 规范

编辑 `web/` 下任何界面（含 `web/concept/`）时先读本 skill，并使用 [`web/concept/tokens.css`](../../../web/concept/tokens.css) 中的 CSS 变量，禁止临时发明色值或字体。

## 文案原则（强制）

- **只显示对玩家有信息量的内容。** 不展示开发备注、指令名映射、空话标题、解释「本面板整合了某某」一类元叙述。
- 第二人称对自己：`look me` / 仪容用 **「你」**，不用「他/她」。
- 数值含义必须可读：双数要标明含义（如「当前 / 先天」），禁止无标签的 `18 / 16`。
- 不要把 `hp`、`score`、`skills`、`dazuo` 等 MUD 命令名露给玩家。

反例（禁止）：

- 「整合传统指令于一处」
- 「【侠客行个人档案】」
- 角标 `look me` / `cmd-tag`

## 字体

| Token | 用途 |
|-------|------|
| `--font-body` | 正文、描写、列表（Noto Serif SC） |
| `--font-display` | 房间名、面板标题、主按钮（ZCOOL XiaoWei） |
| `--font-mono` | 仅调试；玩家 UI 默认不用 |

禁止 Inter / Roboto / Arial / system-ui 作为游戏主字体。

## 基础色板

墨色武侠：深底 + 宣纸字色；点缀只用玉色与朱砂。禁止紫渐变、霓虹光、奶油陶土海报风。

| Token | 含义 |
|-------|------|
| `--ink` | 主背景 |
| `--ink-lift` | 浮层/面板底 |
| `--paper` | 主文字 |
| `--paper-dim` | 次要文字、描写 |
| `--jade` / `--jade-bright` | 安全、出口、进行中、可前往 |
| `--cinnabar` | 危险、战斗、强调句 |
| `--line` / `--line-strong` | 分割线 |

## 属性色（气血 / 档案）

有上限的资源用进度条；无上限的标量只用数字：

| 属性 | Token | 展示 |
|------|-------|------|
| 气 / 精 / 精力 / 内力 / 食物 / 饮水 | `--stat-*` | 色条 + `当前/上限` |
| 经验 | `--stat-exp` | 纯数字（无上限） |
| 潜能 | `--stat-potential` | 纯数字（Web 按标量展示；勿做进度条） |
| 神 / 阅历 | `--stat-shen` 等 | 纯数字 |
| 膂力 / 悟性 / 根骨 / 身法 | `--attr-*` | **当前 / 先天** |

档案还需展示：**攻击**、**防御**（由武功与装备推算）、杀敌与死亡次数。

## 武功

- **已装备**（`enable`/`jifa` 的 skill_map）：按槽位展示内功/兵器/轻功/招架等及有效等级。
- 列表项：名称 + 精通色 + `LvN` + 细进度条（进度 = learned / (等级+1)²）。不要写「升级」「还差 xx」。
- 已装备的功夫用 `□` 标记。

门类色：`--skill-force` / `--skill-weapon` / `--skill-dodge` / `--skill-parry` / `--skill-knowledge` / `--skill-misc`  
精通程度色（低→高）：`--mastery-1` … `--mastery-6`（初学乍练 → 深不可测）

## 出口布局

平面方位用九宫格：**上北下南左西右东**，斜向占四角（东北/西北/东南/西南）。无出口的格子**留空不显示方位字**，避免像可前往方向一样误导。  
**上 / 下 / 进 / 出 / 其它**无法放入平面的出口，竖列在九宫格**右侧**，按钮尺寸与方位格一致。

## 地图

地图浮层分 **区域**（当前城/楼层）与 **世界**（参照 `doc/help/map_all` 总图排布的地标点）。

## 修炼

打坐 / 吐纳 / 练功进行中需滚动展示叙事文案（对齐 mud 的 `message_vision` / `write`），不能只有进度条。  
挂机时若气/精/精力/内力不足，助手应**原地调息等待恢复后自动续练**（状态文案如「调息中 · 气 xx%」），不要空转刷失败，也不要自动跑去睡房。

打坐/吐纳可选「接近满」或「按次数」；练功必须先选择一项**已激发的具体功夫**，按成功次数停止。所有次数输入默认 1；练功不消耗潜能，禁止显示「练到潜能耗尽」。

人物面板「请教」选定功夫后，可选**学习次数**（默认 1）或**学到潜能耗尽**。精不足/授业者疲劳时原地等待；潜能耗尽、达到授业者水平、实战经验不足或授业者离场时明确停止。

手机端顶栏菜单提供「修炼」「动手」入口（与桌面侧栏一致）。

## 浮层与布局

- 角色 / 修炼 / 动手 / 地图等底部浮层：**统一固定高度**（如 `min(620px, 78dvh)`），顶栏与操作按钮不滚，中间 `sheet-scroll` 滚动，避免切 Tab 高度跳动。
- 场景内**只保留一套出口入口**（方向 + 地点名），不再并列「小地图方位条 + 出口列表」。区域总览「地图」为出口区标题行右上角的轻量横向按钮（含楼层）。
- 主界面**场景在上、见闻在下**同页展示，高度约 **55:45**（场景略高），各自独立滚动；见闻默认贴底显示最新消息（上翻后可点「最新」回跟）。
- 顶栏角色条只显示名字与气血条，**不**展示攻防；攻防仅在角色「档案」中查看。

## 物品色（行囊）


| 类型 | Token |
|------|-------|
| 兵器 | `--item-weapon` |
| 防具 / 衣物 | `--item-armor` |
| 食物 / 饮品 | `--item-food` |
| 金钱 | `--item-money` |
| 药品 | `--item-drug` |
| 杂物 | `--item-misc` |
| 已装备标记 | `--item-equipped` |

## 布局与交互（已定方向）

- 游戏中不展示「侠客行」大标题；品牌仅用于启动/登录页。
- 无命令行输入；走动 = 点出口 → 预览 → 确认前往。
- 顶栏角色条打开角色面板；顶栏「菜单」收纳存档 / 帮助 / 退出；地图在场景出口标题行右上角打开区域图（含楼层）。
- 底栏仅保留非对象向高频动作（环顾 / 修炼 / 动手）。
- NPC 人物面板按「常用 / 往来 / 交手」分组：查看、打听、请教为常用；拜师仅对有门派传承的 NPC 显示，货品仅对商人显示；切磋与致命攻击必须分开，攻击使用朱砂危险色并收进折叠区。

## 选择控件（强制）

玩家可见界面**禁止使用原生 `<select>`**（系统弹出框字体/背景与墨色武侠风格冲突）。

| 场景 | 做法 |
|------|------|
| 少量互斥选项（发言方式、性别、低血处置、匹配方式） | `ChoiceRow` 分段按钮（`.choice-row`），选中态玉色描边 |
| 身边人物 / 目标列表 | `.chips` + `.chip`（NPC 用 `.chip.npc`，选中加 `.on`） |
| 自由文本（传音 ID、发言内容） | 墨底输入框 / 多行框，玉色 focus 边 |

共用组件：[`web/app/src/components/ChoiceRow.tsx`](../../../web/app/src/components/ChoiceRow.tsx)。桌面规则编辑器等附属界面同样遵循，勿再引入原生下拉。

## 检查清单

改 UI 前自问：

1. 这段文案删掉后玩家是否丢信息？若否 → 删。
2. 颜色是否来自 `tokens.css`？若否 → 改用 token。
3. 对自己的叙述是否用「你」？
4. 双数值是否已标明「当前 / 先天」或其它明确含义？
5. 互斥选择是否用了 `ChoiceRow` / chip，而不是原生 `<select>`？
